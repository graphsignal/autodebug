**Findings**:
- Scheduler overhead (1.07ms/step) is comparable to model compute (0.66ms/step), dominating TBOT
- `num_continuous_decode_steps=1` causes 1 scheduler round-trip per token; raising it reduces per-token scheduler cost
- `enable_mixed_chunk=False` means ongoing decode steps fully pause when a new prefill arrives; enabling it allows chunked prefill to overlap with decode, reducing TTFT under load
- Model compute is very fast (0.5B on H100); bottleneck is scheduling overhead and IPC

**Planned changes**:
1. `--num-continuous-decode-steps 8` — run 8 decode steps per scheduler round-trip, amortizing 1ms overhead over 8 tokens (effective per-token overhead drops ~7×)
2. `--enable-mixed-chunk` — overlap chunked prefill with decode batch, improving TTFT under concurrent load
3. `--schedule-policy lpm` — longest-prefix-match scheduling to maximize future prefix cache reuse
